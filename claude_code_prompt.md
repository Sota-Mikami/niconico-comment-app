# ニコニコ風Slackコメント表示アプリ 実装プロンプト（Claude Code向け）

このドキュメントは、Claude Codeを使って「ニコニコ動画風のSlackコメント表示デスクトップアプリ（透過・クリックスルー対応）」を実装するための要件定義および実装ステップです。
Claude Codeにこのファイルを読み込ませて、ステップバイステップで実装を依頼してください。

---

## 1. アプリケーション概要
Zoomの画面共有機能を利用しつつ、プレゼンテーション画面（Keynote, Google Slidesなど全画面）の上に、特定のSlackチャンネルから取得したイベント（新規コメント）を右から左へ横流し（ニコニコ動画風）で表示するデスクトップアプリケーション。

## 2. 技術スタック
- **デスクトップアプリ基盤**: Electron (Node.js)
- **フロントエンド（描画）**: React + TypeScript + Vite
- **Slack連携**: `@slack/bolt` （Socket Modeを利用）
- **スタイリング**: Vanilla CSS または TailwindCSS（お好みで選定可、アニメーションはCSSキーフレームやTailwindのユーティリティで実装）

## 3. 主要な機能と要件
### 3.1 Electron (メインプロセス)
- ウィンドウは全画面で起動し、背景を完全に透過させる (`transparent: true`, `frame: false`)。
- 常に最前面に表示する (`alwaysOnTop: true`)。
- マウスイベントを無効化し、クリック等の操作を背後のアプリケーション（プレゼンソフト）に透過させる (`win.setIgnoreMouseEvents(true, { forward: true })`)。
- Mac環境 (macOS) で安定して透過・最前面表示・クリックスルーが行えること。

### 3.2 Slackイベント連携 (メインプロセス)
- `@slack/bolt` を使用し、Slack Appの「App-Level Token (xapp-)」を利用した **Socket Mode** で接続する。
- 起動時に設定（Bot Token (xoxb-)、App-Level Token、監視対象チャンネルID）を環境変数（`.env`）から読み込む。
- 対象チャンネルに投稿された新しいメッセージ (`message` イベント) を監視し、テキストデータをIPC通信でレンダラープロセス（フロントエンド）に送信する。

### 3.3 フロントエンドレンダリング (レンダラープロセス/React)
- IPC経由で受け取ったコメント文字列を、右端から左端へ一定の速度で横に流して表示する。
- **レイアウト・アニメーション最適化**:
  - コメント同士が重ならないように、画面を縦にいくつかの「レーン（行）」に分割する。
  - 新しいコメントが来た際、空いている（一番右端に後続がない）レーンを計算してそこに配置する。
  - CSSの `transform: translateX()` と `transition` (もしくは `animation`) を用いてGPUアクセラレーションを効かせ、カクつきを防ぐ。
  - 画面左端に消えたコメントのDOM要素は適宜削除・クリーンアップし、メモリリークを防ぐ。
- **見た目の調整**: 大きめのフォントサイズ、読みやすい太字、白文字＋黒い縁取り（`text-shadow` 等）など、背景のスライドの色に依存しない視認性の高いスタイル設定。

## 4. Claude Codeへの指示（実装ステップ）

以下のステップに沿って実装を進めてください。1つのステップが完了して正常に動作することを確認してから次のステップへ進んでください。

### Step 1: プロジェクトのセットアップ
Viteを使ってElectron + React + TypeScriptのプロジェクト基盤をこのディレクトリにセットアップしてください。（必要であれば `electron-vite` などのボイラープレートを利用しても構いません）。
セットアップ完了後、`npm run dev`等で起動し、Hello Worldが画面の中央に表示されることを確認してください。

### Step 2: ウィンドウの透過・最前面・クリックスルーパーミッション設定
`main.ts` (または `main.js`) を編集し、Electronのブラウザウィンドウを以下のように設定してください。
- ウィンドウサイズを画面いっぱいに広げる（Fullscreen または WorkAreaSize いっぱい）
- `transparent: true`, `frame: false`, `hasShadow: false`, `alwaysOnTop: true` を付与
- `win.setIgnoreMouseEvents(true, { forward: true })` を設定
実装完了後、アプリを起動し、背景が透過して裏側のウィンドウ（ブラウザなど）を操作できることを確認してください。

### Step 3: Slack Socket Modeの実装
1. `npm install @slack/bolt dotenv` を実行し依存関係を追加してください。
2. メインプロセス側でSlack Appのインスタンスを作成し、Socket Modeで起動する処理を実装してください。`.env`のスタブも用意してください。
3. 指定したチャンネルの `message` イベントを受け取り、メインプロセスのログ (`console.log`) に出力できることを確認してください。
4. 受け取ったメッセージテキストを IPC通信 (`webContents.send()`) でフロントエンドに送信する仕組みを構築してください。

### Step 4: React側でのコメント受信とアニメーションコンポーネント実装
1. React側で、IPC通信から送られてくるメッセージを受信するフック (`useEffect`) を実装してください。
2. 受信したメッセージごとに固有のIDを振り、状態（配列）に追加してください。
3. 各メッセージを画面右端（`100vw` または `left: 100%`）に配置し、CSSアニメーション（`@keyframes`等）で画面左端（`-100%` など）まで等速で移動させるコンポーネントを作成してください。
4. 移動が完了したコンポーネント（一定時間経過後またはアニメーション完了イベント）は状態配列から削除し、クリーンアップするロジックを実装してください。

### Step 5: コメント同士の衝突回避（レーン管理アルゴリズム）の実装
1. 画面の縦幅を基本フォントサイズ（+マージン）で割り、複数の「レーン」を定義してください。
2. 新しいコメントが流れる際、ランダムに配置するのではなく「直近のコメントと重ならないレーン」を順番に探して割り当てるアルゴリズムを実装してください。

### Step 6: 最終的なスタイリングと動作確認
フロントエンドのCSSを調整し、「ニコニコ動画」のような白文字＋黒縁取り等の見やすいスタイルを適用してください。全体を通して仕様通りに動作するか実際に動かして確認してください。
